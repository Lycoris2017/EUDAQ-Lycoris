<p>The user's manual is provided as LaTeX source files in the repository;
to generate the pdf on Linux/OSX, make sure that you have installed ImageMagick, then follow these steps:</p>
<pre><code>cd build
cmake -DEUDAQ_BUILD_MANUAL=ON ..
make pdf
</code></pre>
<p>The manual can then be found in <code>./doc/manual/EUDAQUserManual.pdf</code>.</p>
<p>To generate the doxygen documentation, make sure that you have installed doxygen and run</p>
<pre><code>make doxygen
</code></pre>
<p>in the <code>build</code> directory after CMake. The resulting HTML files are stored in <code>../doc/doxygen/html</code>.</p>

<p>Install Qt5 or later, e.g. by using MacPorts (<a href="http://www.macports.org/" rel="nofollow">http://www.macports.org/</a>): <code>sudo port install qt5-mac-devel</code></p>
<ol start="2">
<li>Configuring and installation/compiling EUDAQ</li>
</ol>
<hr>
<p>cmake will configure the installation and prepare the makefiles. It searches for all the required files. It has to be executed in the <code>build</code> folder, however the relevant CMakeLists.txt is in the main level, thus, the command is <code>cmake ..</code>. If cmake is successful, EUDAQ can be installed. Variables set are cached in CMakeCache.txt and will again be taken into account at the next cmake run.</p>
<p>cmake has several options to activate or deactivate programs which will be built. The main library (libeuaq_core.so) is always built, while the rest of the package is optional. Defaults are to build the main executables and
(if Qt is found) the GUI application. Disable this behavior by setting
e.g. <code>-DEUDAQ_BUILD_GUI=OFF</code> (disabling GUI executables) or enable
e.g. producers using <code>-DEUDAQ_LIBRARY_BUILD_LCIO=ON</code> to enable setting up the
configuration and compilation environment of LCIO extenstion support and
executables. More options:
<a href="https://telescopes.desy.de/EUDAQ#Cmake_options" rel="nofollow">https://telescopes.desy.de/EUDAQ#Cmake_options</a></p>
<p>If cmake is not successful and complains about something is missing, it is recommended to clean the <code>build</code> folder, before a new try, e.g. .</p>
<p>If problems occur during installation, please have a look in the issues, if a similiar problem already occured. If not, feel free to create a new ticket: <a href="https://github.com/eudaq/eudaq/issues">https://github.com/eudaq/eudaq/issues</a></p>

<h3><a id="user-content-windows-1" class="anchor" aria-hidden="true" href="#windows-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Windows</h3>
<p>Start the Visual Studio "Developer Command Prompt" from the Start Menu entries for Visual Studio (Tools subfolder) which opens a <code>cmd.exe</code> session with the necessary environment variables already set. If your Qt installation path has not been added to the global %PATH% variable, you need to execute the "qtenv2.bat" batch file in the Qt folder, e.g.</p>
<pre><code>C:\Qt\Qt5.1.1\5.1.1\msvc2013\bin\qtenv2.bat
</code></pre>
<p>(Replace "5.1.1" with the version string of your Qt installation.)</p>
<p>Go to the EUDAQ folder and configure:</p>
<pre><code>cd c:\[...]\eudaq\build
cmake ..
</code></pre>
<p>(This generates the VS project files.)</p>
<p>Installing into eudaq\bin:</p>
<pre><code>cmake --build . --target install --config Release
</code></pre>
<p>This will compile the main library and the GUI; for the remaining processors, please check the individual documentation.</p>
